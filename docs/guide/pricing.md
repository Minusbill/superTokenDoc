# 分组与计价说明

本页解释两件事：

1. **模型怎么计费**
2. **为什么有“分组倍率”**，以及该怎么选分组更省钱

## 计费说明

### 什么是 Token？

Token 是模型处理文本的基本单位。粗略理解：

- 1 个英文单词约等于 1 到 2 tokens
- 1 个中文字符约等于 1.5 到 2 tokens

### 如何计费？

费用 = (输入 tokens × 输入价格 + 输出 tokens × 输出价格) × 分组倍率

**示例**：

- 输入：500 tokens
- 输出：300 tokens
- 模型：Claude-xx（输入 $1.75/M，输出 $14/M）
- 分组：Claude-Mixed
- 费用：(500 × 1.75 + 300 × 14) / 1,000,000 × 0.6 = $0.00306

## 分组说明

SuperToken 提供多种令牌分组，每种分组有不同的特性和计费倍率：

| 分组名称 | 倍率 | 特性说明 |
| --- | --- | --- |
| **ClaudeCodeMAX** | 1.5x | MAX 号池，限 CC 使用，支持 1M 上下文与高缓存 |
| **Claude-Mixed** | 0.5x | 混合账号池，高缓存命中，支持 200K 上下文 |
| **Codex** | 0.2x | Team、Pro 账号池，必须流式返回，适合在 Codex 内使用 |
| **Gemini** | 1.1x | gemini-cli 与日常对话均可使用 |

### 分组选择建议

- **追求质量**：优先选择 CC 专用或 Claude-Mixed
- **成本优先**：优先选择 Codex 分组
- **特定工具**：ClaudeCode 分组推荐在 Claude Code 内使用，Codex 分组推荐在 Codex 工具内使用。对应软件更容易命中缓存，降低计费；如果在第三方软件中使用高缓存分组，可能因未命中缓存而产生更高费用

## 节省成本的技巧

### 1. 选择合适的模型

- **简单任务**：优先使用轻量模型
- **复杂任务**：优先使用 sonnet 级别模型
- **最高质量**：按需使用 opus、xhigh 系列

使用小技巧：高质量模型价格通常更高，可以把复杂规划交给更强模型，把具体开发交给成本更低的模型完成。

### 3. 优化使用方式

- **控制输出长度**：让工具少输出一点
- **提示词更具体**：减少反复来回
- **能复用上下文就复用**：减少重复输入，在对应工具中使用时，更容易命中上下文缓存，进一步降低费用

### 4. 费用监控

控制台一般会提供：

- 当前余额
- 今日 / 本月消费
- 分组使用统计
- 模型使用分布

## 常见问题

**Q: 如何选择合适的分组？**  
A: 写代码且想控制成本时可优先考虑 Codex 分组；如果是在 Claude Code 内使用，也可以考虑 Claude-Mixed；如果更关注高上下文和更强体验，可考虑 ClaudeCodeMAX。

**Q: Claude Code MAX 分组只能在 Claude Code 工具中使用吗？**  
A: Claude Code MAX 分组是为 Claude Code 工具优化的，命中缓存后更省，也更适合大上下文场景。
